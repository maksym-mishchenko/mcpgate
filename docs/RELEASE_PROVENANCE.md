# Release provenance and checksums

mcpgate release artifacts are built by the repository release workflow when a `vX.Y.Z` tag is pushed.

## What is published

Each release should include:

- OS/architecture archives generated by GoReleaser.
- `checksums.txt` with SHA-256 checksums for release archives.
- GitHub artifact attestations for the archives and checksum file.

Do not trust binaries shared outside the GitHub release page unless their checksum and attestation match the release.

## Verify checksums

Download the archive for your platform and `checksums.txt` from the GitHub release, then run:

```bash
shasum -a 256 -c checksums.txt --ignore-missing
```

The selected archive should report `OK`.

## Verify provenance

Install the GitHub CLI and verify the artifact attestation against this repository:

```bash
gh attestation verify ./mcpgate_<version>_<os>_<arch>.tar.gz \
  --repo maksym-mishchenko/mcpgate
```

For Windows archives:

```bash
gh attestation verify ./mcpgate_<version>_windows_<arch>.zip \
  --repo maksym-mishchenko/mcpgate
```

The attestation should identify the `Release` workflow in this repository as the build source.

## Maintainer release checklist

1. Confirm the tag points at the intended commit on `main`.
2. Confirm CI passed before pushing the tag.
3. Confirm the release contains archives, `checksums.txt`, and attestations.
4. Confirm a downloaded archive passes checksum verification.
5. Confirm `gh attestation verify` passes for at least one archive.
