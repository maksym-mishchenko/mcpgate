# Code of Conduct

mcpgate is a small security-focused open-source project. Keep discussion professional, practical, and safe for maintainers and contributors.

## Expected behavior

- Be respectful and assume good faith.
- Keep feedback specific, actionable, and grounded in project behavior.
- Disclose security-sensitive findings privately through GitHub Security Advisories.
- Avoid sharing secrets, private prompts, production audit logs, or other sensitive data in public issues and pull requests.
- Respect maintainer decisions about project scope, release timing, and security posture.

## Unacceptable behavior

- Harassment, insults, threats, or discriminatory language.
- Publicly posting exploit details for an undisclosed vulnerability.
- Spam, flooding, or repeated off-topic comments.
- Social engineering, credential requests, or attempts to bypass repository safety processes.
- Publishing another person's private information without permission.

## Enforcement

Maintainers may edit or remove comments, close issues, block users, or report abuse to GitHub when behavior undermines project safety or contributor trust.

To report conduct concerns, contact the maintainer through GitHub. For vulnerabilities, use the private disclosure process in [SECURITY.md](SECURITY.md).
